# Техническое задание — Анкета КЦ

> **Статус:** актуальный документ. Устаревшая версия удалена.
> Файл `TZ.md` (PascalCase) — дублирует этот документ и будет удалён.

---

## Проект

Локальное приложение Bitrix24 для анкеты колл-центра. Открывается как вкладка (таб) в карточке лида CRM. Интерфейс — iframe поверх Bitrix24 REST API и BX24 JS SDK. Backend — PHP.

---

## Утверждённый технологический стек

| Направление | Решение | Примечание |
|---|---|---|
| Frontend styles | **Tailwind CSS 4** | CDN-сборка, utility-first |
| UI components | **Flowbite 2** | Готовые компоненты поверх Tailwind |
| CRM / portal | **Bitrix24 REST API + BX24 JS SDK** | Локальное приложение, `CRM_LEAD_DETAIL_TAB` |
| Backend | **PHP** | `config.php`, `install.php`, `proxy.php`, `uninstall.php` |

**Правило:** стек зафиксирован. При каждой новой итерации UI не проектируется заново — интерфейс развивается внутри одного утверждённого стека и набора компонентов.

---

## Требования к frontend

### Общие правила

- Любой новый экран или переработка строится на **Tailwind CSS 4**.
- Стандартные UI-элементы (кнопки, формы, карточки, вкладки, алерты, спиннеры) берутся из **Flowbite**.
- Самописные CSS-файлы (`tokens.css`, `style.css`) **не подключаются** в `index.php` — они помечены как устаревшие и оставлены только в истории git.
- Кастомные стили допустимы только точечно для Bitrix24 iframe/embed-особенностей (например, `overflow: hidden` на `body`).
- Новый дизайн **не проектируется с нуля** при каждой итерации.

### Layout

Основной экран — двухколоночный:
- **Левая колонка (55%)** — форма анкеты по блокам с прогресс-баром.
- **Правая колонка (45%)** — расписание менеджеров + вкладка "Фишки" для будущих инструментов.

Правая панель расширяема: скоринг, документы, история изменений, подсказки для менеджера.

### Блоки формы (левая колонка)

| # | Блок | Поля |
|---|---|---|
| 1 | Финансовые данные | ФИО (auto), сумма долга, платёж, офиц. доход, неофиц. доход, зарплатная карта |
| 2 | Кредитная история | Кредиторы, залог, просрочки, ФССП, имущество, сделки |
| 3 | Личные данные | Место работы, семейное положение, дети, совм. имущество, судимости, ООО, ИП |
| 4 | Заметки менеджера | Исключение из КМ, основная боль, возражения, доп. комментарий |
| 5 | Запись к специалисту | Менеджер, время, ID события (через расписание правой панели) |

### Компоненты Flowbite

| Компонент | Применение |
|---|---|
| `Tabs` | Переключение правой панели (расписание / фишки) |
| `Card` | Блоки формы |
| `Button` | Кнопки действий (primary, secondary) |
| `Input`, `Select`, `Textarea` | Поля формы |
| `Alert` | Ошибки и сообщения об успехе |
| `Spinner` | Состояние загрузки |
| `Badge` | Статусы слотов расписания |
| `Collapse` | Сворачивание блоков формы |

## Требования к интеграции (Bitrix24)

| Операция | Метод |
|---|---|
| Получение данных лида | `crm.lead.get` |
| Получение списка пользователей | `user.get` (отдел КМ, `SALES_DEPT_ID`) |
| Сохранение анкеты | `crm.lead.update` (26 × `UF_CRM_KC_*` полей) |
| Запись в таймлайн | `crm.timeline.comment.add` |
| Получение расписания | `calendar.event.get` |
| Бронирование слота | `calendar.event.add` |
| Уведомление менеджера | `im.notify` |

Backend-часть (PHP) не меняет стек:
- `config.php` — константы портала.
- `install.php` — создание UF-полей при установке.
- `proxy.php` — серверный прокси для REST API.
- `uninstall.php` — очистка при удалении.
