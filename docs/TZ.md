# Техническое задание — Bitrix24 Local App «Анкета МКЦ + МП»

## Общие сведения

| Параметр | Значение |
|---|---|
| Тип приложения | Локальное приложение Bitrix24 (`CRM_LEAD_DETAIL_TAB`) |
| Роли | МКЦ (колл-центр), МП (менеджер продаж) |
| Встраивание | iframe во вкладке карточки лида CRM |
| Репозиторий | `petriaevden-cmd/bitrix24-local-app-questionnaire` |

---

## Технологический стек

| Слой | Технология | Версия / CDN |
|---|---|---|
| Frontend | Tailwind CSS 4 | `cdn.jsdelivr.net/npm/@tailwindcss/browser@4` |
| UI-компоненты | Flowbite 2 | `cdnjs.cloudflare.com/ajax/libs/flowbite/2.3.0/` |
| Интеграции | Bitrix24 REST API + BX24 JS SDK | `{portal}/bitrix/js/rest/bx24.js` |
| Backend | PHP | Нативный PHP без фреймворков |

### Правила стека (не менять)

- Самописные CSS-файлы (`tokens.css`, `style.css`) **не подключаются** и не создаются.
- Кастомные стили допустимы только в `<style>` внутри `index.php` для iframe-reset (сброс отступов браузера в embed-контексте Bitrix24).
- Новые компоненты UI строятся **только на Tailwind 4 + Flowbite 2**.
- JavaScript остаётся нативным (ES2020+), без npm-сборки и фреймворков.
- Backend — только PHP: `config.php`, `install.php`, `uninstall.php`, `proxy.php`.

## Архитектура интерфейса

### Макет

Двухколоночный split-layout, фиксированная высота = 100vh:

```
┌─────────────────────────────────────────────────────┐
│  Шапка: Название + Лид + Пользователь BX24           │
├────────────────────────┬────────────────────────────┤
│   ЛЕВАЯ (55%)          │   ПРАВАЯ (45%)              │
│   Анкета               │   Расписание / Фишки        │
│   - Прогресс-бар       │   - Навигация по дням        │
│   - 5 блоков полей     │   - Кнопка «Обновить»        │
│   - Кнопка «Сохранить» │   - Таблица слотов МП        │
└────────────────────────┴────────────────────────────┘
```

### Блоки анкеты (левая колонка)

| № | Название | ID контейнера | Колонок |
|---|---|---|---|
| 1 | Персональные данные | `#personal-body` | 2 |
| 2 | Финансовые данные | `#finance-body` | 2 |
| 3 | Кредитная история | `#credit-body` | 2 |
| 4 | Заметки менеджера | `#manager-body` | 1 |
| 5 | Запись на встречу | `#booking-body` | — |

### Расписание (правая колонка)

- Вкладки: **Расписание** (активна) / **Фишки** (заглушка).
- Навигация: кнопки «Пред. день» / «След. день» + текущая дата.
- Таблица слотов: строки = МП, столбцы = часовые слоты.
- Каждый слот отображает **два времени**: время МП + время клиента (timezone-aware).
- Цвет слота: 🟢 свободен, 🔘 занят.

## Логика обновления расписания

### Триггеры загрузки слотов

| Событие | Действие | Файл |
|---|---|---|
| Первая загрузка анкеты | `initCalendar()` → `loadAllSlots()` | `app.js` |
| После бронирования слота | `loadAllSlots()` | `calendar.js` |
| Клик «Обновить расписание» | `loadAllSlots()` вручную | `index.php` (inline JS) |

### Автоматический polling — **ОТКЛЮЧЁН**

- `polling.js` содержит пустой `startPolling()` (no-op).
- `setInterval` **не используется**.
- Повторных фоновых запросов расписания нет.
- Обоснование: лишняя нагрузка на API Bitrix24, ненужное мерцание таблицы.

### Кнопка «Обновить расписание»

- Расположение: правая колонка, под навигацией по дням, над таблицей слотов.
- При клике: иконка анимируется (`animate-spin`), кнопка блокируется на 1.5 сек.
- Вызывает `loadAllSlots()` напрямую.

## Логика часового пояса

1. МКЦ/МП выбирает **город клиента** в блоке 1 анкеты (поле `UF_CRM_KC_CLIENT_CITY`).
2. Из встроенного словаря городов определяется `utcOffset` клиента.
3. Таблица слотов отображает **две колонки времён**: время МП (UTC МП) и время клиента (UTC клиента).
4. Если рабочие часы МП не пересекаются с доступным временем клиента сегодня — автоматически переключается на следующий рабочий день.
5. Поле города является обязательным (`required`) — форма не сохраняется без его заполнения.

## Валидация формы

| Поле | Проверка | Поведение при ошибке |
|---|---|---|
| `UF_CRM_KC_CLIENT_CITY` | Непустое | Красная рамка, inline-сообщение, фокус на поле |

- Валидация выполняется в `form.js` перед вызовом `crm.lead.update`.
- При вводе/изменении города ошибка сбрасывается автоматически.
