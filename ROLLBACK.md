# ROLLBACK — Инструкция по откату изменений

Этот документ описывает, **как вернуть работу приложения в iframe Битрикс24** после временного перевода на вебхук (для тестирования вне iframe).

## TL;DR (быстрый откат за 2 минуты)

1. Откройте `anketa-kc/index.php` → найдите блок `=== NEW: WEBHOOK MODE ===` в `<head>`.
2. Замените значение флага `window.APP_USE_WEBHOOK = true;` → `window.APP_USE_WEBHOOK = false;`
3. Раскомментируйте два тега `<script>` в блоке `=== ROLLBACK START (legacy BX24 SDK load) ===`.
4. Откройте `anketa-kc/assets/app.js` → первые 30 строк: убедитесь, что `leadId = null` (уже сделано — SDK заполнит автоматически).
5. Перезалейте файлы на сервер, где установлено локальное приложение.

Всё. Приложение снова работает как обычное iframe-приложение Битрикс24.

---

## Что именно было изменено

Чтобы можно было тестировать приложение **вне iframe Битрикс24** (в обычном браузере, с данными из входящего вебхука), были внесены **3 обратимых изменения**. Каждое помечено маркерами `ROLLBACK START` / `ROLLBACK END` или `NEW` в исходниках.

### Изменение 1 — отключена загрузка BX24 JS SDK

**Файл:** `anketa-kc/index.php`  
**Строки:** блок `<!-- === ROLLBACK START (legacy BX24 SDK load) === -->`

Закомментированы два `<script>`-тега, которые подключали оригинальный BX24 SDK с сервера портала:

```html
<!--
<script src="//api.bitrix24.tech/api/v1/"></script>
<script src="//<?= $portalHost ?>/bitrix/js/rest/bx24/bx24.min.js"></script>
-->
```

**Откат:** снимите HTML-комментарий `<!-- ... -->`, обрамляющий эти две строки.

### Изменение 2 — добавлен shim-клиент для вебхука

**Файлы:**
- Новый — `anketa-kc/assets/webhook-client.js` (≈330 строк).
- Изменён `anketa-kc/index.php`: блок `<!-- === NEW: WEBHOOK MODE === -->`.

Создан модуль `webhook-client.js`, который подменяет глобальный объект `window.BX24` на shim, делающий запросы через входящий вебхук (`https://crm.yurclick.com/rest/6/m1umtpppnvj21gud/`). Интерфейс shim 1-в-1 совпадает с BX24 SDK (`callMethod`, `callBatch`, `placement.info`, `init`).

Shim активируется только при `window.APP_USE_WEBHOOK === true` в `index.php`.

**Откат — два варианта:**

- **Мягкий (оставить файлы, выключить режим)** — установите в `index.php`:
  ```js
  window.APP_USE_WEBHOOK = false;
  ```
  И раскомментируйте тега BX24 SDK (см. Изменение 1). Shim при этом остаётся в репозитории, но не подключается (либо подключается, но ничего не делает, т.к. BX24 уже установлен SDK).

- **Жёсткий (удалить новый код)** — удалите из `index.php` весь блок между `<!-- === NEW: WEBHOOK MODE === -->` и `<!-- === END: WEBHOOK MODE === -->`, а также файл `anketa-kc/assets/webhook-client.js`. Раскомментируйте оригинальные теги BX24 SDK.

### Изменение 3 — закомментированы hardcoded-дефолты в `app.js`

**Файл:** `anketa-kc/assets/app.js`, блок `// === ROLLBACK START (hardcoded debug defaults) ===`

До изменений в файле были жёстко прописаны `leadId=59466`, `currentUser=14`, `CURRENT_USERNAME='Петряев Денис'` — для ручной отладки. После изменений они заменены на `null` / `''`, а старые значения сохранены в комментарии.

В iframe-режиме эти переменные заполняются автоматически из `BX24.placement.info()` и `user.current`.

**Откат:** если вам снова понадобятся именно эти дефолтные значения — удалите строки с `= null` / `= ''` и раскомментируйте исходные значения, сохранённые в комментарии блока.

### Изменение 4 — исправлен синтаксический баг в `form.js` (не обратимо нужно)

**Файл:** `anketa-kc/assets/form.js`, функция `validateForm()`.

В оригинальном коде была синтаксическая ошибка: отключённый блок валидации был закомментирован через `/* */` неправильно — закрывающие скобки `}` оказались вне комментария. Это ломало загрузку всего `form.js`, из-за чего поля формы не отрисовывались.

Исправлено: весь отключённый код корректно заключён в `/* ... */`, поведение функции не изменилось (она по-прежнему возвращает `true` и пропускает валидацию).

**Откат не рекомендуется** — баг был реальным и мешал работе приложения даже в iframe-режиме. Но при желании можно вернуть исходный текст из Git-истории:

```bash
git log --oneline -- anketa-kc/assets/form.js
git show <старый-коммит>:anketa-kc/assets/form.js > anketa-kc/assets/form.js
```

## Проверка после отката

1. Установите приложение в Битрикс24 (`install.php`).
2. Откройте любой лид в CRM → вкладка «Анкета КЦ и МП».
3. В консоли браузера (F12) **не должно быть** строки `[webhook-client] Режим разработки...`.
4. Должны корректно отработать:
   - автозаполнение ФИО из поля лида,
   - отображение имени текущего пользователя в шапке,
   - загрузка расписания всех МП через `calendar.accessibility.get`,
   - сохранение анкеты (запись полей `UF_CRM_KC_*` + комментарий в таймлайн),
   - бронирование слота через `bizproc.workflow.start`.

Если что-то не работает — проверьте:
- `PORTAL_URL` в `config.php` ≠ `crm.yurclick.com` (указан ли ваш реальный портал),
- `BP_TEMPLATE_ID`, `SALES_DEPT_ID` — соответствуют вашей инсталляции,
- `MP_CONFIG` в `assets/mp-config.js` — ID сотрудников заменены на реальные (не плейсхолдеры 1–11).

## Git-стратегия (рекомендуется)

Если вы хотите сохранить возможность откатываться через Git:

```bash
# Фиксируем состояние «вебхук-режим»
git add -A
git commit -m "dev: переключение на вебхук для тестирования вне iframe"

# Создаём тег для быстрого возврата
git tag rollback-point-before-iframe-restore

# Когда будете возвращаться к iframe — откатитесь к этому тегу
# git reset --hard rollback-point-before-iframe-restore
```
