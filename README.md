# Анкета КЦ — локальное приложение Bitrix24

Виджет для менеджеров колл-центра (МКЦ) и менеджеров по продажам (МП).  
Встраивается внутрь лида отдельной вкладкой и позволяет во время звонка заполнить анкету клиента и сразу же записать его на встречу с МП.

---

## Назначение

Когда МКЦ или МП обрабатывает новый лид, он переключается на вкладку виджета и:

1. **Заполняет анкету** (левая половина экрана) — данные о клиенте: финансовое положение, кредитная история, заметки менеджера.
2. **Выбирает слот встречи** (правая половина экрана) — видит расписание всех МП на текущий день и записывает клиента на свободное время в один клик.
3. **Сохраняет** — поля анкеты записываются в пользовательские поля лида (`UF_CRM_KC_*`), данные о встрече запускают бизнес-процесс для автоматической записи в календарь МП.

---

## Расположение в Bitrix24

Тип плейсмента: **`CRM_LEAD_DETAIL_TAB`**  
Виджет открывается как отдельная вкладка внутри карточки лида. ID лида передаётся автоматически через `BX24.placement.info().options.ID`.

## Как работает анкета

### Загрузка данных
При открытии виджет получает данные лида через `crm.lead.get` и текущего пользователя через `user.current` (один `BX24.callBatch`). Поля анкеты предзаполняются из лида — если МКЦ уже заполнял анкету ранее, данные будут на месте.

### Поля анкеты → поля лида

| Поле лида (`UF_CRM_KC_*`) | Тип | Описание |
|---|---|---|
| `KC_FULLNAME` | string | ФИО (авто из `NAME`+`LASTNAME`+`SECONDNAME`) |
| `KC_CLIENT_CITY` | string | Город клиента — **обязательное**, используется для TZ в расписании |
| `KC_WORKPLACE` | string | Место работы |
| `KC_MARITAL_STATUS` | enumeration | Семейное положение |
| `KC_CHILDREN` | enumeration | Дети (0 / 1 / 2 / 3+) |
| `KC_JOINT_PROPERTY` | enumeration | Совместное имущество |
| `KC_CRIMINAL` | enumeration | Судимости |
| `KC_OOO` | enumeration | ООО |
| `KC_IP` | enumeration | ИП |
| `KC_DEBT_TOTAL` | integer | Общая сумма долга, ₽ |
| `KC_MONTHLY_PAYMENT` | integer | Ежемесячный платёж, ₽ |
| `KC_INCOME_OFFICIAL` | enumeration | Официальный доход |
| `KC_INCOME_UNOFFICIAL` | integer | Неофициальный доход, ₽ |
| `KC_SALARY_CARD` | enumeration | Зарплатная карта (Сбер / другой / нет) |
| `KC_CREDITORS` | string | Кредиторы (банки, МФО) |
| `KC_COLLATERAL` | enumeration | Залог |
| `KC_OVERDUE` | string | Просрочки |
| `KC_FSSP` | enumeration | ФССП |
| `KC_PROPERTY` | enumeration | Имущество |
| `KC_DEALS` | enumeration | Сделки |
| `KC_KM_EXCLUSION` | string | Исключение из КМ |
| `KC_MAIN_PAIN` | string | Основная боль клиента |
| `KC_OBJECTIONS` | string | Возражения |
| `KC_EXTRA_COMMENT` | string | Доп. комментарий |
| `KC_BOOKED_MANAGER` | employee | ID менеджера (заполняется при бронировании) |
| `KC_BOOKED_TIME` | datetime | Время записи (заполняется при бронировании) |
| `KC_BOOKED_EVENT_ID` | integer | ID события календаря (заполняется при бронировании) |

Сохранение происходит через `crm.lead.update`. После успешного сохранения в таймлайн лида добавляется комментарий с кратким резюме (`crm.timeline.comment.add`).

### Город → часовой пояс
Поле «Город клиента» является обязательным. Введённый город ищется в справочнике `CITIES_TZ` (900+ городов России). Найденное UTC-смещение используется в расписании для отображения времени встречи в часовом поясе клиента — МКЦ видит сразу оба времени: время МП и время клиента.

## Как работает расписание

### Загрузка занятости
При инициализации и при смене дня виджет запрашивает занятость каждого активного МП через `calendar.accessibility.get`. Занятыми считаются события со статусом `busy` или `absent`.

### Свободные слоты
Для каждого МП генерируются слоты внутри его рабочего времени (`workStart`–`workEnd`) с шагом `APP_CONFIG.slotMin` минут (по умолчанию 60). Слоты фильтруются по:
- пересечению с занятыми событиями
- уже прошедшему времени
- рабочим часам клиента (`clientHrMin`–`clientHrMax`) — если город клиента известен

### Отображение
Таблица: строки = МП, столбцы = уникальные UTC-слоты дня.  
В заголовке каждой колонки — два времени: время МП и (синим) время клиента.  
Зелёная кнопка — свободный слот. Красный кружок — занято. Серая полоска — вне рабочего времени МП.

### Бронирование (текущее состояние — заглушка)
При нажатии на слот показывается блок подтверждения. После подтверждения:
1. `calendar.event.add` — создаёт событие в календаре МП (**временная реализация**, см. раздел "Что не реализовано")
2. `crm.lead.update` — сохраняет `KC_BOOKED_MANAGER`, `KC_BOOKED_TIME`, `KC_BOOKED_EVENT_ID` в лид
3. `crm.timeline.comment.add` — добавляет запись о встрече в таймлайн лида
4. Расписание обновляется

### Целевая реализация бронирования через бизнес-процесс
По архитектуре бронирование должно выполняться через запуск бизнес-процесса Bitrix24, который автоматически создаёт событие в календаре МП, уведомляет участников и совершает другие автоматические действия.

Запуск БП делается через `bizproc.workflow.start`:

```js
BX24.callMethod('bizproc.workflow.start', {
  TEMPLATE_ID: APP_CONFIG.bookingWorkflowId, // ID шаблона БП
  DOCUMENT_ID: ['crm', 'CCrmDocumentLead', 'LEAD_' + leadId],
  PARAMETERS: {
    // параметры определяются шаблоном БП, например:
    managerId:  'user_' + mpUserId,  // ID менеджера (формат user_N)
    slotTime:   fmtBxUTC(slot.utcMs) // время слота в формате Bitrix24
  }
}, function(result) {
  if (result.error()) showError('Ошибка запуска БП: ' + result.error());
  else { /* успех */ }
});
```

> **Важно:** запуск `bizproc.workflow.start` доступен только администраторам и требует scope `bizproc` в манифесте. Конкретные параметры (`TEMPLATE_ID`, состав `PARAMETERS`) определяются разработчиком БП.

## Конфигурация МП (`mp-config.js`)

```js
const MP_CONFIG = {
  mp1: {
    bitrixUserId: 123,        // числовой ID пользователя в Bitrix24
    name:         'Иван Иванов',
    city:         'Москва',   // для определения UTC-смещения МП
    workStart:    '09:00',
    workEnd:      '18:00',
    workDays:     [1,2,3,4,5], // 1=пн, 7=вс (по JS Date.getDay(): 0=вс, 1=пн...)
    slotMinutes:  60,
    active:       true
  },
  // ...
};
```

> **Ограничение:** сейчас МП добавляются вручную редактированием `mp-config.js`. В будущем список МП должен подтягиваться динамически из Bitrix24 (например через `user.get` с фильтром по должности или отделу).

## Установка

1. Разместите файлы папки `anketa-kc/` на HTTPS-сервере.
2. Заполните `config.php` — укажите `PORTAL_URL`, `APP_CLIENT_ID`, `APP_CLIENT_SECRET`.
3. Обновите `manifest.json` — замените `install_url` и `handler_url` на реальные адреса.
4. Создайте в Bitrix24 пользовательские поля лида (`UF_CRM_KC_*`) согласно таблице выше.
5. Установите приложение через Bitrix24 → Приложения → Разработчикам.
